# Swiftfin 弹幕功能测试指南

## 🧪 测试步骤

### 1. 编译测试
```bash
# 在 Xcode 中编译项目
# 确保没有编译错误
```

### 2. 基础功能测试

#### 2.1 弹幕按钮测试
1. 启动应用并播放任意视频
2. 点击播放器右上角的弹幕按钮 (💬)
3. 验证按钮状态变化：
   - 未启用：灰色 `bubble.left` 图标
   - 已启用：紫色 `bubble.left.fill` 图标

#### 2.2 弹幕设置测试
1. 进入 设置 → 视频播放器
2. 找到 "弹幕设置" 部分
3. 测试各项设置：
   - 启用/禁用弹幕
   - 调整不透明度 (0.1-1.0)
   - 调整字体大小 (12-24)
   - 调整滚动速度 (0.5-2.0x)
   - 调整最大显示数量 (5-50)
   - 切换弹幕类型显示开关

### 3. 弹幕数据测试

#### 3.1 配置测试服务器
在 `DanmakuService.swift` 中修改服务器地址：
```swift
private let baseURL = "你的弹幕服务器地址"
```

#### 3.2 测试弹幕加载
1. 播放视频
2. 启用弹幕功能
3. 观察控制台日志：
   ```
   弹幕数据解析成功: 获取到X条弹幕
   成功加载弹幕段 0-30: X条弹幕
   ```

### 4. 性能测试

#### 4.1 内存使用测试
1. 使用 Xcode Instruments 监控内存使用
2. 播放长视频并启用弹幕
3. 观察内存是否稳定，无明显泄漏

#### 4.2 渲染性能测试
1. 设置最大弹幕数量为 50
2. 播放弹幕密集的视频段
3. 观察播放是否流畅，无卡顿

### 5. 边界情况测试

#### 5.1 网络异常测试
1. 断开网络连接
2. 播放视频并启用弹幕
3. 验证应用不会崩溃，有适当的错误处理

#### 5.2 数据异常测试
1. 配置无效的弹幕服务器地址
2. 验证错误处理和日志输出

## 🐛 常见问题排查

### 问题1：弹幕按钮不显示
**可能原因**：
- 弹幕按钮未添加到 barActionButtons 配置中

**解决方案**：
1. 进入 设置 → 视频播放器 → 按钮 → 操作栏按钮
2. 添加 "弹幕" 按钮

### 问题2：弹幕不显示
**可能原因**：
- 弹幕服务器地址配置错误
- 网络连接问题
- 弹幕数据格式不兼容

**排查步骤**：
1. 检查控制台日志
2. 验证服务器地址和网络连接
3. 检查弹幕数据格式

### 问题3：性能问题
**可能原因**：
- 弹幕数量过多
- 渲染优化不足

**解决方案**：
1. 降低最大显示数量
2. 检查对象池是否正常工作
3. 使用 Instruments 分析性能瓶颈

## 🔧 调试技巧

### 启用详细日志
在 `DanmakuService.swift` 和 `DanmakuViewModel.swift` 中已经集成了详细的日志输出，可以通过控制台查看弹幕加载和处理过程。

### 使用断点调试
在以下关键位置设置断点：
- `DanmakuService.fetchDanmakuSegment` - 弹幕数据获取
- `DanmakuViewModel.updateDanmakus` - 弹幕更新逻辑
- `DanmakuRenderer.addDanmakuItem` - 弹幕渲染

### 性能分析
使用 Xcode Instruments 的以下工具：
- Time Profiler - 分析 CPU 使用
- Allocations - 分析内存使用
- Core Animation - 分析渲染性能
